# Pendapatan (Income)

## 1. Definisi

**Pendapatan (Income)** adalah kenaikan manfaat ekonomi selama satu periode akuntansi dalam bentuk arus masuk atau peningkatan aset, atau penurunan liabilitas, yang mengakibatkan kenaikan ekuitas, tidak berasal dari kontribusi pemilik.  

Pendapatan merupakan **akun nominal (temporary account)** — artinya saldo akun ini akan **ditutup di akhir periode** ke akun laba ditahan (`equity.retained_earnings.current_year`).  
Akun ini bersaldo **kredit**, dan posisinya berada dalam **laporan laba rugi** (statement of profit or loss).

---

## 2. Ciri-Ciri Akun Pendapatan

| Karakteristik | Penjelasan |
| :-- | :-- |
| **Jenis Akun** | Nominal (Temporary Account) |
| **Saldo Normal** | Kredit |
| **Posisi Laporan Keuangan** | Laporan Laba Rugi |
| **Bertambah Ketika** | Entitas menghasilkan pendapatan dari penjualan barang atau jasa |
| **Berkurang Ketika** | Terjadi retur penjualan, potongan penjualan, atau koreksi pendapatan |
| **Tujuan Pengukuran** | Menunjukkan kinerja dan sumber penghasilan entitas selama periode tertentu |
| **Contoh Umum** | Pendapatan Penjualan, Pendapatan Jasa, Pendapatan Bunga, Pendapatan Sewa |

---

## 3. Struktur Path Akun Pendapatan

Struktur penamaan akun pendapatan menggunakan format:

```

income.[kategori].[subkategori].[detail].[spesifik]

```

Untuk pelaporan pajak dan analitik, akun pendapatan dapat dilengkapi dengan **tag khusus** di bagian `[spesifik]`:
- `.final` → pendapatan yang dikenakan PPh final.  
- `.non_final` → pendapatan biasa yang dikenakan PPh non-final (melalui laba rugi fiskal).

---

### 3.1 Kategori Utama Pendapatan

| Kategori | Deskripsi | Contoh Path |
| :-- | :-- | :-- |
| **sales** | Pendapatan utama dari aktivitas penjualan barang atau jasa | `income.sales.product.retail.non_final` |
| **service** | Pendapatan dari pemberian jasa profesional | `income.service.consulting.non_final` |
| **rental** | Pendapatan dari sewa aset | `income.rental.property.final` |
| **interest** | Pendapatan bunga atau hasil keuangan lainnya | `income.interest.deposit.final` |
| **commission** | Pendapatan komisi dari transaksi pihak ketiga | `income.commission.agent.non_final` |
| **other** | Pendapatan di luar kegiatan utama (miscellaneous income) | `income.other.gain_on_sale_asset.non_final` |

---

### 3.2 Contoh Path Akun Pendapatan Lengkap

```

income.sales.product.retail.non_final
income.sales.product.wholesale.non_final
income.service.consulting.non_final
income.service.training.non_final
income.rental.property.final
income.interest.deposit.final
income.other.gain_on_sale_asset.non_final
income.commission.agent.non_final

```

Setiap path di atas bersifat **deskriptif dan bermakna semantik**.  
Contoh:
> `income.rental.property.final` → Pendapatan sewa properti yang dikenakan PPh final.

---

## 4. Klasifikasi Pajak: Final vs Non-Final

Dalam konteks perpajakan Indonesia, tidak semua pendapatan dikenakan pajak dengan cara yang sama.  
Standar *Less Summer* menandai perbedaan tersebut secara eksplisit dalam struktur path akun.

### 4.1 Pendapatan Final (`.final`)
Merupakan penghasilan yang dikenakan **PPh Final**, artinya pajak sudah dipotong atau dibayar secara final dan tidak diperhitungkan lagi dalam laba rugi fiskal.

**Contoh:**
- Pendapatan bunga deposito → `income.interest.deposit.final`  
- Pendapatan sewa tanah/bangunan → `income.rental.property.final`

### 4.2 Pendapatan Non-Final (`.non_final`)
Merupakan penghasilan operasional biasa yang dikenakan **PPh non-final** (dilaporkan dalam laba rugi fiskal).  
Pendapatan ini digunakan untuk menghitung **penghasilan kena pajak (PKP)** di akhir periode.

**Contoh:**
- Penjualan jasa konsultasi → `income.service.consulting.non_final`  
- Penjualan barang dagang → `income.sales.product.retail.non_final`

Dengan penggunaan tag `.final` dan `.non_final`, sistem pelaporan pajak atau ERP dapat **memisahkan secara otomatis** pendapatan fiskal dan non-fiskal tanpa konfigurasi manual tambahan.

---

## 5. Contoh Transaksi

### 5.1 Penjualan Jasa Konsultasi (Non-Final)
> Pendapatan jasa konsultasi sebesar Rp25.000.000 secara kredit.

| Tanggal | Keterangan | Path Akun | Debit | Kredit |
| :-- | :-- | :-- | :-- | :-- |
| 2025-10-10 | Piutang Usaha | `assets.receivable.customer` | 25.000.000 | |
| 2025-10-10 | Pendapatan Jasa Konsultasi | `income.service.consulting.non_final` | | 25.000.000 |

---

### 5.2 Pendapatan Sewa Properti (Final)
> Pendapatan sewa gedung sebesar Rp10.000.000 yang dikenakan PPh Final.

| Tanggal | Keterangan | Path Akun | Debit | Kredit |
| :-- | :-- | :-- | :-- | :-- |
| 2025-10-20 | Kas di Bank Mandiri | `assets.cash.bank.mandiri` | 10.000.000 | |
| 2025-10-20 | Pendapatan Sewa Gedung | `income.rental.property.final` | | 10.000.000 |

---

## 6. Relasi dengan Jenis Akun Lain

| Relasi | Hubungan yang Terjadi |
| :-- | :-- |
| **Aset** | Pendapatan menambah aset, umumnya berupa kas atau piutang. |
| **Liabilitas** | Pendapatan diterima di muka menimbulkan liabilitas sementara (`liabilities.unearned.income`). |
| **Ekuitas** | Pendapatan meningkatkan ekuitas melalui laba ditahan setelah periode ditutup. |
| **Beban (Expenses)** | Pendapatan dan beban berinteraksi untuk menentukan laba bersih. |

Pendapatan berperan sebagai **komponen utama laba rugi** dan menjadi faktor penentu pertumbuhan ekuitas dari periode ke periode.

---

## 7. Catatan Tambahan

### 7.1 Retur dan Potongan Penjualan
Setiap penurunan pendapatan dapat dicatat dengan akun kontra dalam kelompok pendapatan:
```

income.sales.return.product.retail.non_final
income.sales.discount.product.wholesale.non_final

```
Akun kontra bersaldo **debit**, tetapi tetap diklasifikasikan dalam kelompok pendapatan.

### 7.2 Pendapatan yang Ditangguhkan
Pendapatan yang diterima sebelum jasa diberikan dicatat sementara sebagai liabilitas:
```

liabilities.unearned.income.subscription

````
dan dipindahkan ke akun pendapatan (`income.service.subscription.non_final`) setelah jasa diberikan.

### 7.3 Integrasi Sistem
- Path `.final` dan `.non_final` memungkinkan sistem ERP atau modul pajak mengenali jenis penghasilan tanpa filter tambahan.  
- Analitik pendapatan dapat dikelompokkan otomatis per sumber (produk, jasa, sewa, bunga, dll.) melalui hierarki path.

Contoh query:
```sql
SELECT SUM(amount)
FROM journal_entries
WHERE account_path LIKE 'income.%.non_final';
````

---

## 8. Kesimpulan

Pendapatan adalah **cerminan aktivitas ekonomi utama entitas** selama periode tertentu, menjadi dasar bagi pengukuran kinerja dan evaluasi profitabilitas.
Sebagai **akun nominal (temporary)**, pendapatan menutup siklus operasional dengan memindahkan saldo ke laba ditahan di akhir periode.

Struktur path seperti:

```
income.service.consulting.non_final
income.rental.property.final
income.interest.deposit.final
```

menjadikan setiap akun pendapatan **deskriptif, konsisten, dan siap otomatisasi**, baik untuk pelaporan komersial maupun fiskal.

Melalui *Standar Penamaan Akun Less Summer*, pendapatan tidak hanya tercatat sebagai angka, tetapi juga **bercerita tentang asal dan sifatnya** — menjembatani pencatatan akuntansi dengan transparansi pelaporan pajak dan analitik bisnis modern.
